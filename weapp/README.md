# 微信小游戏最小接入说明

本目录提供**最小示例配置**，用于说明 Cocos Creator 构建到微信小游戏时常见的补充文件与注意事项。

## 建议流程

1. 在 Cocos Creator 3.8.6 中选择构建平台为“微信小游戏”。
2. 先执行一次编辑器构建，让编辑器输出标准小游戏构建目录。
3. 根据实际小游戏构建产物，按需合并本目录中的配置示例。

## 文件说明

- `game.json`：微信小游戏基础配置示例。

## 注意事项

- 实际入口文件通常由 Cocos 构建产物生成，请不要脱离编辑器产物手工拼完整运行时。
- 如需分包、横竖屏、子包预下载、开放数据域，请在实际构建目录中继续补充。
- 平台相关调用建议统一收敛到 `assets/scripts/platform/` 下适配层。

## 当前边界

- 本目录仅提供微信小游戏侧的**最小示例配置**与补充说明，不构成正式主循环验收入口。
- 是否存在拖拽、关卡、合成、收集等功能代码，不应仅凭本目录推导为“小游戏端已整体完成”。
- 当前仓库稳定、仓库级、可直接执行的正式入口包括：`npm run type-check` 与 `npm run verify:chapter1-drag-loop`。
- 其余运行、构建、真机接线仍需结合 Cocos 编辑器产物、`weapp/` 配置示例与实际环境。
- 如需理解历史显眼验证/报告/接线文件为何不再保留在根目录，请查看 `archive/history/root-signals/` 与根目录统一说明文档。
